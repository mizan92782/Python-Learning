OrderedDict is used when we need a dictionary that remembers the order in which items are inserted.

🔥 Why it is useful?
Maintains insertion order explicitly (especially important in older Python versions < 3.7)
Useful when order of data matters (like logs, steps, sequences)
Helps in algorithms where order tracking is required (LRU cache, parsing, etc.)
Allows reordering of elements using methods like move_to_end()

from collections import Counter

## 📌 What is Counter?
- Counter is used to count distinct elements in a container
- It creates a frequency mapping of elements
- It is a subclass of dictionary designed for counting objects

---

## 📥 Import
```python
from collections import Counter
📊 Function Table
Function / Operation	Why it is used
Counter(iterable)	Create frequency count from an iterable
update(iterable)	Add new elements and update counts
elements()	Return all elements expanded by their count
most_common(n)	Get top n most frequent elements
ctr[key] += n	Manually increase count of a key
ctr[key] -= n	Manually decrease count of a key
subtract(iterable)	Subtract counts using another iterable or Counter
ctr1 + ctr2	Add two Counters (merge frequencies)
ctr1 - ctr2	Subtract counts (keep only positive results)
ctr1 & ctr2	Intersection (minimum of common counts)
`ctr1	ctr2`
📌 Why use Counter() instead of a normal dictionary?
Counter and dictionary are similar, but Counter provides additional functionality specifically for counting elements
Returns 0 for missing keys, whereas a dictionary raises a KeyError
Designed specifically for counting hashable items, while dictionary is general-purpose
Provides built-in methods like most_common(), elements(), update() and subtract() for easier frequency operations
Supports arithmetic operations such as +, -, &, | for direct manipulation
Requires less manual code compared to dictionary-based counting

📌 Does normal dict keep insertion order?

👉 Yes — modern Python dictionaries DO keep insertion order.

Since Python 3.7+, the built-in dict preserves the order in which items are inserted.

🔥 Example:
d = {}
d["a"] = 1
d["b"] = 2
d["c"] = 3

print(d)
Output:
{'a': 1, 'b': 2, 'c': 3}

Order is maintained ✅


⚠️ Important Note
Python 3.7+ → dict keeps insertion order (official language feature)
Python 3.6 → order is preserved but not guaranteed (implementation detail)
Python 3.5 or older → no order guarantee ❌
📌 Then why OrderedDict?

OrderedDict is still useful because:

It has extra methods like move_to_end()
Better for LRU cache implementation
Useful when you need order manipulation, not just preservation
🧠 Summary:

👉 Normal dict = keeps insertion order (Python 3.7+)
👉 OrderedDict = keeps order + advanced order control features
